0.3
Roadmap

0.3.1: Introduction

ဒီ boot camp မှာ မင်းတို့ ဘာတွေသင်ယူရမလဲ၊ ဘာတွေလုပ်ဆောင်နိုင်မလဲဆိုတာကို လေ့လာကြည့်ရအောင်။ ဒါ့အပြင် သင်တန်းတစ်လျှောက် မင်းကို အထောက်အကူပေးမယ့် resources တွေကိုလည်း တစ်ခါတည်း ကြည့်သွားမှာဖြစ်ပါတယ်။

0.3.2: What You’ll Learn in Coding Boot Camp

ဒီ Boot Camp ကို အဆင့် ၃ ဆင့်နဲ့ ညီတူညီမျှ ခွဲထားပါတယ် - Foundation၊ Technical နဲ့ Performance တို့ပါ။ သင်တန်းတစ်လျှောက်မှာ Virtual Classes တွေတက်ရမယ်၊ ကိုယ်တိုင်လေ့ကျင့်ရမယ်၊ ပြီးတော့ လက်တွေ့နယ်ပယ်က ပြဿနာတွေကို ဖြေရှင်းပေးနိုင်မယ့် Web Applications တွေကို တည်ဆောက်သွားရမှာပါ။

Phase 1: Foundation (အခြေခံအဆင့်)

ဒီအဆင့်မှာ Web Development ရဲ့ အခြေခံတွေဖြစ်တဲ့ Front-end ပိုင်း၊ Command-line အခြေခံတွေနဲ့ APIs တွေအကြောင်းကို သင်ရပါမယ်။

  • HTML နှင့် CSS
  • Git
  • JavaScript
  • Third-party APIs (jQuery, Bootstrap)
  • Server-side APIs, AJAX နှင့် JSON

ပထမဆုံး Project မှာ Front-end ပိုင်းကို ကိုယ်တိုင်ရေးဆွဲပြီး တခြားသူတွေရဲ့ Servers နဲ့ APIs တွေကို ချိတ်ဆက်အသုံးပြုရမှာပါ။

Phase 2: Technical (နည်းပညာအဆင့်)

ဒီအဆင့်မှာတော့ Full-stack Web Application တစ်ခုလုံးကို တည်ဆောက်နိုင်ဖို့ လိုအပ်တဲ့ Back-end နည်းပညာတွေ၊ Servers တွေနဲ့ Databases တွေကို သင်ယူရမှာပါ။

  • Node.js
  • Object-oriented programming (OOP)
  • Express.js servers
  • MySQL
  • ORM (Sequelize)
  • Model-view-controller (MVC) framework (Handlebars.js)
  • Unit testing
  • Agile methodology
  • Heroku or Firebase

ဒုတိယ Project မှာတော့ သင်ထားတဲ့ Front-end အရည်အချင်းနဲ့ Server Code ရေးနိုင်တဲ့ အရည်အချင်းကို ပေါင်းစပ်ပြီး Back-end နဲ့ တိုက်ရိုက်အလုပ်လုပ်နိုင်မယ့် Web Application ကို အဖွဲ့လိုက် တည်ဆောက်ရမှာပါ။

Phase 3: Performance (စွမ်းဆောင်ရည်အဆင့်)

ဒီနောက်ဆုံးအဆင့်မှာတော့ ကိုယ်ရေးထားတဲ့ Application တွေကို ပိုမြန်အောင်၊ ပိုကောင်းအောင် ဘယ်လို Optimize လုပ်မလဲဆိုတာနဲ့ အလုပ်ခွင်ဝင်ဖို့အတွက် ပြင်ဆင်တာတွေကို သင်ရမှာပါ။

  • Computer science အခြေခံများ
  • MongoDB နှင့် NoSQL
  • Progressive web apps (PWAs)
  • React
  • MERN Stack
  • Portfolios (အလုပ်လျှောက်ဖို့ ကိုယ်ပိုင်လက်ရာမှတ်တမ်း)

သင်တန်းရဲ့ အပိတ် Capstone Project အနေနဲ့ သင်ခဲ့သမျှ Languages နဲ့ Libraries အစုံသုံးပြီး Full-stack Application အကြီးစားတစ်ခုကို အဖွဲ့လိုက် တည်ဆောက်ရမှာပါ။

0.3.3: Boot Camp Expectations and Guidelines

ဒီသင်တန်းမှာ လိုက်နာရမယ့် ယေဘုယျ လမ်းညွှန်ချက်တွေနဲ့ မျှော်မှန်းချက်တွေပါ။ အသေးစိတ်ကိုတော့ Syllabus မှာ သေချာဖတ်ပေးပါ။

  • သင်တန်းအတွက် အချိန်သီးသန့်ပေးဖို့ အရေးကြီးပါတယ်။ တစ်ပတ်ကို စုစုပေါင်း ၂၀ ကနေ ၃၀ နာရီ လောက် အချိန်ပေးဖို့ စီစဉ်ထားပါ။
  • Academic Honesty (ကျောင်းစည်းကမ်းအတိုင်း ရိုးသားမှု) ရှိဖို့ လိုပါတယ်။ Challenge assignments တွေနဲ့ Projects တွေဟာ ကိုယ်တိုင် (ဒါမှမဟုတ် ကိုယ့်အဖွဲ့) ရဲ့ Original work (ကိုယ်တိုင်လုပ်ထားတဲ့ လက်ရာ) ပဲ ဖြစ်ရပါမယ်။
  • အမှတ်အပြည့်ရဖို့ Assignments နဲ့ Projects အားလုံးကို တင်ရပါမယ်။ Challenge assignment ၂ ခုထက်ပိုပြီး ပျက်ကွက်လို့ မရပါဘူး။
  • Tutoring: မိနစ် ၅၀ ကြာ one-on-one (တစ်ဦးချင်း) သင်တန်းတောင်းလို့ရပါတယ်။ အကူအညီတောင်းတဲ့အခါ screenshots တွေနဲ့ ကိုယ်တိုင် ဘယ်လိုဖြေရှင်းဖို့ ကြိုးစားခဲ့သလဲဆိုတာကို အသေးစိတ် ပြောပြပေးဖို့ လိုပါတယ်။
  • Career Services: အလုပ်အကိုင်ဈေးကွက်မှာ အပြိုင်အဆိုင်ဖြစ်အောင် (employer-competitive ဖြစ်အောင်) resources တွေ၊ workshop တွေနဲ့ one-on-one coaching တွေပေးပြီး ကူညီသွားမှာပါ။
  • ဆရာတွေ၊ TAs တွေနဲ့ အတန်းဖော်တွေကြားမှာ ရိုသေပြီး အားပေးတဲ့ စကားလုံး၊ လေယူလေသိမ်းတွေကို သုံးပေးပါ။
  • ဒီသင်တန်းကို WCAG 2.0 စံနှုန်းတွေနဲ့အညီ လူတိုင်းအဆင်ပြေပြေ လေ့လာနိုင်အောင် (accessible ဖြစ်အောင်) စီစဉ်ထားပါတယ်။ အကူအညီလိုရင် SSM ကို ဆက်သွယ်ပါ။

0.3.4: How to Succeed in the Coding Boot Camp

ဒီ Coding Boot Camp ကို တက်ဖို့ ဆုံးဖြတ်လိုက်တဲ့အတွက် ဂုဏ်ယူပါတယ်။ မင်းရဲ့ Attitude (စိတ်နေစိတ်ထား) က အောင်မြင်ဖို့အတွက် အဓိကသော့ချက်ပါပဲ။ ဒီခရီးစဉ်မှာ အောင်မြင်ပြီး အကျိုးရှိစေဖို့ အောက်ပါ Tips လေးတွေကို လိုက်နာကြည့်ပါ -

၁။ စိတ်ပျက်စရာတွေကို ကျော်လွှားပါ (Overcome Frustration)

ဒီသင်တန်းမှာ သင်ရမယ့်အရာတွေက မင်းကို စိန်ခေါ်ပါလိမ့်မယ်။ တစ်ခါတလေ စိတ်ညစ်ရနိုင်ပေမဲ့ "ငါတိုးတက်လာမှာပါ" လို့ ကိုယ့်ကိုယ်ကိုယ် အားပေးပါ။ အပတ်တိုင်းကို အသစ်တွေသင်ယူဖို့ အခွင့်အရေးတစ်ခုလို သဘောထားပါ။ နောက် ၆ လကြာတဲ့အခါ မင်း ဘယ်လောက်တောင် ခရီးပေါက်ခဲ့ပြီလဲဆိုတာ ပြန်ကြည့်ပြီး အံ့သြရပါလိမ့်မယ်။

၂။ ကိုယ့်သင်ယူမှုကိုယ် တာဝန်ယူပါ (Take Charge of Your Learning)

မင်း ဘယ်လောက် အားထုတ်သလဲဆိုတဲ့အပေါ်မှာပဲ ရလဒ်က မူတည်ပါတယ်။ Coding ဆိုတာ လက်တွေ့လေ့ကျင့်မှု (Practice)၊ စမ်းသပ်မှု (Experimentation) နဲ့ တိုးတက်ချင်စိတ် ရှိဖို့ လိုပါတယ်။ Support Team က လမ်းညွှန်ပေးမှာဖြစ်ပေမဲ့ အားထုတ်မှုကတော့ မင်းကိုယ်တိုင်လုပ်ရမှာပါ။

၃။ အချိန်ပေးပါ (Put In the Time)

ဒါဟာ ကိုယ့်ကိုယ်ကိုယ် ရင်းနှီးမြှုပ်နှံတာပါ။ Assignments တွေနဲ့ Projects တွေကို ဘယ်အချိန်၊ ဘယ်နေရာမှာ လုပ်မယ်ဆိုတာ Schedule နဲ့ အစီအစဉ် (Action Plan) သေချာဆွဲထားပါ။ စာသင်ချိန်မှာ အေးဆေးတဲ့နေရာမှာ ထိုင်ပါ။ အလုပ်လုပ်နေရင်း ခဏနားဖို့နဲ့ အိပ်ရေးဝဝအိပ်ဖို့လည်း မမေ့ပါနဲ့။

၄။ Google အသုံးပြုတဲ့ အတတ်ပညာကို လေ့ကျင့်ပါ (Refine the Art of Using Google)

Web Development ဆိုတာ အချက်အလက်တွေကို အလွတ်ကျက်ရတာ မဟုတ်ပါဘူး။ လိုအပ်တာကို ရှာဖွေပြီး စုစည်းတတ်ဖို့ပါ။ ပြဿနာတစ်ခုကို Google မှာ Research လုပ်ပြီး ဖြေရှင်းနိုင်စွမ်းက အရေးကြီးဆုံး Skill တစ်ခုဖြစ်ပါတယ်။

၅။ အစီအစဉ်ဆွဲပါ (Make a Plan)

Bootcamp Spot မှာ အပတ်စဉ် တနင်္ဂနွေနေ့တိုင်း သင်ခန်းစာအသစ်တွေ ပွင့်ပါတယ်။ အဲ့ဒီအချိန်မှာ Weekly Challenge ရဲ့ လိုအပ်ချက်တွေကို ကြိုကြည့်ပြီး တစ်ပတ်စာ အစီအစဉ်ဆွဲထားပါ။ Assignments တွေကို နောက်ဆုံးရက်မတိုင်ခင် အပြီးလုပ်ပါ။

၆။ ကျန်းမာရေး ဂရုစိုက်ပါ (Prioritize Self-Care)

ကွန်ပျူတာရှေ့မှာ အကြာကြီး ထိုင်ရမှာဖြစ်လို့ Posture (ထိုင်ပုံထိုင်နည်း) မှန်ပါစေ၊ အသက်ဝဝရှူပါ။ ခဏခဏ အကြောဆန့်ပေးဖို့နဲ့ ကိုယ်လက်လှုပ်ရှားမှုလုပ်ဖို့ အရေးကြီးပါတယ်။ အခက်အခဲရှိရင် Support Community နဲ့ ချိတ်ဆက်ပြီး အကူအညီယူပါ။

0.3.5: Be Original

ဒီသင်တန်းရဲ့ အဓိကရည်ရွယ်ချက်က အသုံးဝင်တဲ့ Skills တွေရဖို့ပါ။ သင်ခန်းစာတွေ ဘယ်လောက်ပဲ ခက်ခဲပါစေ၊ ဘယ်လောက်ပဲ စိတ်ညစ်စရာကောင်းပါစေ၊ အမြဲတမ်း ကိုယ်တိုင်လုပ်ထားတဲ့ Original work ကိုပဲ တင်ပေးပါ။ အခက်အခဲရှိရင် Tutor တွေကို အကူအညီတောင်းဖို့ ဒါမှမဟုတ် Tutor network ဆီ အီးမေးလ်ပို့ဖို့ ဝန်မလေးပါနဲ့။

Plagiarism (သူများလက်ရာ ခိုးယူခြင်း) ကို လုံးဝခွင့်မပြုပါဘူး။ အကယ်၍ ခိုးချထားတာ တွေ့ရှိရင် အမှတ်မပေးတာ ဒါမှမဟုတ် ဘွဲ့မပေးတာမျိုးအထိ အရေးယူခံရနိုင်ပါတယ်။

What Is Plagiarism? (ခိုးယူခြင်းဆိုတာ ဘာလဲ?)

သူများရဲ့ စာသား၊ စိတ်ကူး (Concepts) ဒါမှမဟုတ် Product တွေကို ကိုးကားချက် (Citation) မပါဘဲ ကိုယ့်လက်ရာလိုမျိုး တင်ပြတာကို ခေါ်တာပါ။ ဥပမာအားဖြင့် -

  • သူများရဲ့ စာသား ဒါမှမဟုတ် ပုံတွေကို မူရင်းဘယ်ကရလဲဆိုတာ မဖော်ပြဘဲ ကူးယူတာ။
  • သူများရဲ့ Ideas တွေကို ကိုယ့်စကားနဲ့ ပြန်ပြောပေမဲ့ မူရင်းဘယ်သူလဲဆိုတာ မပြောတာ။
  • Challenge နဲ့ Project တွေမှာ ဝိုင်းလုပ်ပေးသူတွေကို အသိအမှတ်မပြုတာ။
  • အွန်လိုင်း Blogs၊ Websites တွေက အချက်အလက်တွေကို Credit မပေးဘဲ ယူသုံးတာ။
  • သူများရေးထားတဲ့ Code ကို ယူပြီး ကိုယ့် Code လိုမျိုး တင်လိုက်တာ။

Code နဲ့ ပတ်သက်ရင် - အခြေခံ Code အနည်းငယ် (ဥပမာ- Menu တစ်ခုဆောက်ဖို့ လိုအပ်တဲ့ စာကြောင်းအနည်းငယ်) ကို Manual တွေထဲက ကူးယူတာဟာ ပုံမှန်အားဖြင့် ပြဿနာမရှိပါဘူး။ ဒါပေမဲ့ သိသိသာသာ များပြားတဲ့ Code အပိုင်းအစတွေကို ယူသုံးမယ်ဆိုရင်တော့ Comment ထဲမှာ ဘယ်ကရလဲဆိုတာ သေချာဖော်ပြပေးရပါမယ်။

Tips to Avoid Plagiarism (ရှောင်ကြဉ်ဖို့ နည်းလမ်းများ)

မင်းရဲ့ လက်ရာတွေဟာ Original ဖြစ်နေစေဖို့ အောက်ပါအချက်တွေကို လိုက်နာပါ -

  • Cite All Borrowed Content: သူများဆီက ယူသုံးတဲ့ အကြောင်းအရာ မှန်သမျှကို Quotation marks သုံးပါ ဒါမှမဟုတ် Link ထည့်ပေးပါ။
  • Use Your Own Designs: သူများဆီက Design Inspiration (စိတ်ကူး) ယူလို့ရပေမဲ့ တစ်ပုံစံတည်း ကူးတာမျိုး မလုပ်ပါနဲ့။ ကိုယ်ပိုင်လက်ရာ ဖြစ်အောင် ဖန်တီးပါ။
  • Cite All Borrowed Code: သူများဆီက Code တစ်စိတ်တစ်ပိုင်း ယူသုံးရင် Comment ထဲမှာ ၎င်းရဲ့ Link နဲ့ မူရင်းနေရာကို ထည့်ပေးပါ။ အက်ဆေးရေးသလိုပါပဲ၊ စာကြောင်းအနည်းငယ် ကိုးကားတာ ရပေမဲ့ တစ်ခုလုံး ကူးတာမျိုးတော့ မရပါဘူး။
  • Comment Your Code: ကိုယ့် Code က ဘယ်လိုအလုပ်လုပ်လဲဆိုတာကို Comments သေချာရေးပါ။ ဒါဟာ အလုပ်ခွင်မှာလည်း အရေးကြီးတဲ့ အလေ့အကျင့်ကောင်းတစ်ခုပါ။
  • Commit to GitHub Early and Often: အလုပ်တစ်ခုပြီးတိုင်း ဒါမှမဟုတ် Bug တစ်ခုပြင်ပြီးတိုင်း GitHub ပေါ်ကို ခဏခဏ Commit လုပ်ပါ။ ဒါမှ မင်းရဲ့ Code တဖြည်းဖြည်း တိုးတက်လာပုံ (Evolution) ကို သက်သေပြနိုင်မှာပါ။

သူများလက်ရာကို ကိုယ့်ဟာလိုမျိုး တင်ပြတာဟာ ကျင့်ဝတ်နဲ့ မညီပါဘူး။ အထက်ပါ နည်းလမ်းတွေကို သုံးပြီး ကိုယ်တိုင်ဖန်တီးတဲ့ Full-stack Web Developer ကောင်းတစ်ယောက်ဖြစ်အောင် ကြိုးစားကြရအောင်။

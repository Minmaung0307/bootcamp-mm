Set Up the Project

Introduction

Full-stack web developer တစ်ယောက်ဖြစ်ဖို့ ခရီးစတင်လိုက်တဲ့အတွက် ကြိုဆိုပါတယ်! အခုလာမယ့် တစ်ပတ်အတွင်းမှာ developer တစ်ယောက်ဖြစ်ဖို့ အခြေခံအကျဆုံးနဲ့ အရေးကြီးဆုံးဖြစ်တဲ့ HTML, CSS နဲ့ Git နည်းပညာ သုံးခုကို ကျွမ်းကျင်အောင် သင်ယူရမှာပါ။

ဒီပညာရပ်တွေကို Run Buddy ဆိုတဲ့ ကုမ္ပဏီအတွက် landing page တစ်ခု တည်ဆောက်ရင်းနဲ့ လက်တွေ့လေ့လာသွားမှာပါ။ မင်းကို freelance web developer တစ်ယောက်အနေနဲ့ ငှားထားတယ်လို့ သဘောထားပြီး ဒီ website ကို အလှပဆုံးဖြစ်အောင် ဖန်တီးပေးရမှာ ဖြစ်ပါတယ်။ အပြင်အဆင်ပိုင်း (design) ပြီးသွားရင်တော့ ကုမ္ပဏီရဲ့ developer တွေဆီကို လက်လွှဲပေးရမှာပါ။

ဒီသင်ခန်းစာမှာ ဘာတွေတတ်မြောက်မလဲ?

ပထမဆုံး သင်ခန်းစာမှာတင် HTML code တွေ စရေးနိုင်ဖို့အတွက် အရေးကြီးတဲ့ tools တွေနဲ့ လုပ်ငန်းစဉ်တွေကို လေ့လာရမှာပါ။ အဓိကအားဖြင့် အောက်ပါတို့ကို လုပ်ဆောင်နိုင်မှာ ဖြစ်ပါတယ် -

  • Command line ကို သုံးပြီး folder တွေထဲ သွားလာတာနဲ့ file အသစ်တွေ တည်ဆောက်တာ။
  • HTML document တစ်ခုရဲ့ အခြေခံ structure ကို စနစ်တကျ ချမှတ်တာ။
  • Git ကို သုံးပြီး ကိုယ့်ရဲ့ အလုပ်တွေကို သိမ်းဆည်းတာနဲ့ remote repository ပေါ်ကို push လုပ်တာ။

Preview (အကြိုကြည့်ရှုခြင်း)

ဒီအပတ်မှာ ဘာတွေ တည်ဆောက်မလဲဆိုတာ အကြမ်းဖျင်း ကြည့်လိုက်ရအောင်။ Run Buddy ဒီဇိုင်းအဖွဲ့ကနေ Landing page အတွက် Mock-up (ပုံစံကြမ်း) တစ်ခု ပေးထားပါတယ်။

အရေးကြီးချက်: Run Buddy စာမျက်နှာကို လုပ်နေတဲ့အချိန်မှာ ဒီပုံကို ခဏခဏ ပြန်ကြည့်ရမှာဖြစ်လို့ သင့်ကွန်ပျူတာထဲမှာ Download ဆွဲထားဖို့ အကြံပြုပါတယ်။ ပုံပေါ်မှာ ညာဘက်ထောက် (Right-click) နှိပ်ပြီး "Save Image As" နဲ့ မှတ်မိလွယ်မယ့်နေရာမှာ သိမ်းထားလိုက်ပါ။

သင်ဟာ Developer အသစ်တစ်ယောက်အနေနဲ့ ဒီ Mock-up ကိုကြည့်ပြီး "တော်တော်ခက်မှာပဲ" လို့ ထင်ကောင်းထင်နိုင်ပါတယ်။ စိတ်မပူပါနဲ့၊ ကျွန်တော်တို့ တစ်ဆင့်ချင်းစီ အေးအေးဆေးဆေး သွားကြမှာပါ။ Web development နည်းပညာက အမြဲတမ်း အလျှင်အမြန် ပြောင်းလဲနေတာဖြစ်လို့ ဝါရင့် Developers တွေတောင် Project အသစ်စတိုင်း Tools အသစ်တွေကို ပြန်သင်ယူရလေ့ရှိပါတယ်။ ဒီသင်တန်းမှာ မင်းတို့ တွေ့ရှိလာမှာကတော့ Developer တစ်ယောက်ဖြစ်လာဖို့ဆိုတာ "ဘယ်လိုသင်ယူရမလဲ" ဆိုတာကို သင်ယူခြင်းပဲ ဖြစ်ပါတယ်။

အခု ပထမဆုံးသင်ခန်းစာမှာတော့ အောက်မှာပြထားတဲ့အတိုင်း အခြေခံ Layout ကိုပဲ အရင်ဆုံး တည်ဆောက်မှာ ဖြစ်ပါတယ်။

ဒါကို အောင်အောင်မြင်မြင် လုပ်နိုင်ဖို့ အောက်ပါအဆင့်တွေကို လိုက်နာသွားပါမယ် -

  1. Command line နဲ့ စတင်ခြင်း: Project ရဲ့ File structure ကို တည်ဆောက်ဖို့ အခြေခံ Commands တွေကို လေ့လာပါမယ်။
  2. HTML အနည်းငယ် ထည့်သွင်းခြင်း: Project အတွက် အဓိက HTML file ကို တည်ဆောက်ပြီး အရေးကြီးတဲ့ HTML elements တွေအကြောင်း သင်ယူပါမယ်။
  3. Git ကို အသုံးပြုခြင်း: Git version control tool ရဲ့ အခြေခံတွေကို လေ့လာပြီး ဒီ project အတွက် Repository ဆောက်တာနဲ့ ပထမဆုံး Commits လုပ်တာတွေကို လုပ်ဆောင်ပါမယ်။
  4. စာမျက်နှာကို Outline ဆွဲခြင်း: HTML အကြောင်းကို ပိုနက်နက်ရှိုင်းရှိုင်း လေ့လာပြီး Project အတွက် အခြေခံ Structure ကို စီစဉ်ပါမယ်။
  5. Git workflow ကို ပြန်လည်သုံးသပ်ခြင်း: လုပ်ဆောင်ထားတဲ့ တိုးတက်မှုတွေကို local Git repository ထဲမှာ Commit လုပ်ပါမယ်။
  6. GitHub နဲ့ Publish လုပ်ခြင်း: ကိုယ်တိုင်လုပ်ထားတဲ့ Local repository ကို GitHub Pages ပေါ် တင်ပါမယ်။ ဒါဆိုရင် ကမ္ဘာပေါ်က ဘယ်သူမဆို မင်းရဲ့ website ကို ကြည့်လို့ရသွားပါပြီ။

ကဲ... အခုပဲ Coding စတင်လိုက်ရအောင်!

Get Started with the Command Line

အရင်တုန်းကတော့ ကွန်ပျူတာကို စာသားတွေရိုက်ပြီး ခိုင်းရတဲ့ Command-line interface (CLI) ဒါမှမဟုတ် Terminal နဲ့ပဲ အသုံးပြုခဲ့ကြတာပါ။ အခုခေတ်မှာ Windows နဲ့ macOS တို့လို ပုံတွေ၊ ခလုတ်တွေနဲ့ သုံးရတဲ့ Operating Systems တွေ ပေါ်လာတော့ CLI ကို သုံးစရာမလိုတော့သလို ဖြစ်နေပေမဲ့ Developers တွေကတော့ အလုပ်တော်တော်များများအတွက် CLI ကို အခုထိ သုံးနေရတုန်းပါပဲ။

အခုလောလောဆယ်တော့ ကျွန်တော်တို့ Project ကို Set up လုပ်ဖို့ CLI ကို သုံးသွားပါမယ်။ အဲ့ဒီမတိုင်ခင် ဒါကို ဘယ်လိုဖွင့်ရမလဲဆိုတာ အရင်ကြည့်ရအောင်။

Access the Command Line (Command Line ကို ဖွင့်နည်း)

macOS အသုံးပြုသူများ

macOS မှာ Terminal ဆိုတဲ့ App ပါပြီးသားပါ။ Spotlight Search (စာရိုက်ပြီး ရှာဖွေခြင်း) မှာ "terminal" လို့ ရိုက်ရှာပြီး ဖွင့်နိုင်ပါတယ်။ Spotlight Search ကို ဖွင့်ဖို့ မှန်ဘီလူးပုံလေးကို နှိပ်ပါ ဒါမှမဟုတ် Command+Space ကို နှိပ်ပါ။

Windows အသုံးပြုသူများ

Prework မှာ Git Bash ကို Install လုပ်ခဲ့ပြီးပြီနော်။ Windows စခရင်ရဲ့ အောက်ခြေ ဘယ်ဘက်မှာရှိတဲ့ Search box မှာ "git bash" လို့ ရိုက်ရှာပြီး ဖွင့်လိုက်ပါ။ ဒါမှမဟုတ် Windows+S ကို နှိပ်ပြီး ရှာဖွေနိုင်ပါတယ်။

Use the Command Line (စတင်အသုံးပြုခြင်း)

Command Line ကို ဖွင့်လိုက်တဲ့အခါ စာရိုက်ဖို့ စောင့်နေတဲ့ cursor လေး ပါတဲ့ window တစ်ခု ပေါ်လာပါလိမ့်မယ်။ အဲ့ဒီ window ထဲမှာ pwd လို့ ရိုက်ပြီး Enter ခေါက်လိုက်ပါ။

pwd ဆိုတာ "Print Working Directory" ကို ပြောတာပါ။ Print ဆိုတာက စခရင်ပေါ်မှာ ပြပေးဖို့ဖြစ်ပြီး Working Directory ဆိုတာကတော့ CLI က အခုလက်ရှိ ရောက်နေတဲ့ Folder ကို ပြောတာပါ။ ဒါကြောင့် pwd လို့ ရိုက်တာဟာ "ငါအခု ဘယ်နေရာ ရောက်နေတာလဲ?" လို့ မေးလိုက်တာပါပဲ။

IMPORTANT

ဒီ boot camp မှာ အောင်မြင်ဖို့အတွက် code တွေကို copy/paste မလုပ်ဘဲ တစ်ကြောင်းချင်းစီ ကိုယ်တိုင် လက်နဲ့ ရိုက်ရေးပါ။ ဒါမှသာ ကျွမ်းကျင်တဲ့ coder တစ်ယောက်ဖြစ်ဖို့ လိုအပ်တဲ့ muscle memory ကို အမြန်ဆုံး တည်ဆောက်နိုင်မှာပါ။

Terminal ဒါမှမဟုတ် Git Bash ကို စဖွင့်လိုက်ရင် ပုံမှန်အားဖြင့် ကိုယ့်ရဲ့ User Directory (မိမိအကောင့်အမည်နဲ့ folder) ထဲမှာ ရောက်နေပါလိမ့်မယ်။ pwd လို့ ရိုက်ကြည့်ရင် Windows မှာ /c/Users/your-username လို့ ပြပြီး macOS မှာတော့ /Users/your-username လို့ ပြပါလိမ့်မယ်။

အခု ဒီ Directory ထဲမှာ ဘာတွေရှိလဲဆိုတာ ls လို့ ရိုက်ပြီး ကြည့်ရအောင်။ ဒီ Command က လက်ရှိရောက်နေတဲ့ နေရာက Files တွေနဲ့ Folders တွေကို စာရင်းပြပေးတာပါ။ pwd က "ငါဘယ်မှာလဲ?" လို့ မေးတာဆိုရင် ls က "ဒီမှာ ဘာတွေရှိလဲ?" လို့ မေးတာပါပဲ။

ls ရိုက်လိုက်ရင် Documents, Pictures, Music နဲ့ Desktop စတဲ့ folder တွေကို တွေ့ရပါလိမ့်မယ်။

Nerd Note

ကွန်ပျူတာ စဖွင့်လိုက်ရင် မြင်ရတဲ့ Desktop ဆိုတာ တကယ်တော့ မင်းရဲ့ User Directory ထဲမှာ ရှိတဲ့ Desktop ဆိုတဲ့ Folder လေးပါပဲ။

အခု Desktop ပေါ်ကို CLI ကနေ သွားကြည့်ရအောင်။ အောက်က Commands တွေကို တစ်ကြောင်းချင်း ရိုက်ပြီး Enter ခေါက်ပါ - 

        cd Desktop
        pwd

PAUSE

cd ဆိုတာ ဘာကို ဆိုလိုတာလို့ ထင်လဲ?

အဖြေ - cd ဆိုတာ Change Directory ကို ပြောတာပါ။ Folder တစ်ခုကနေ တစ်ခုကို ကူးပြောင်းဖို့ သုံးပါတယ်။
Show Answer

အခု အောက်ကအတိုင်း ထပ်ရိုက်ကြည့်ပါ။ pwd မှာ ဘာတွေပြလဲဆိုတာ သတိထားကြည့်နော် - 

        cd ..
        pwd
        cd Desktop
        pwd

ဒါကြောင့် cd .. လို့ ရိုက်ရင် အနောက်ကို တစ်ဆင့် (ဒါမှမဟုတ် အပေါ်က Directory ထဲကို) ပြန်သွားတယ်ဆိုတာ မှတ်ထားလိုက်ပါ!

အခု Desktop ပေါ် ပြန်ရောက်နေပြီဆိုရင် ls လို့ ရိုက်ကြည့်ပါ။ ကိုယ့် Desktop ပေါ်မှာ တကယ်ရှိနေတဲ့ ဖိုင်တွေကို မြင်ရတယ်မလား?

အခု Folder အသစ်တစ်ခု ဆောက်ကြည့်ရအောင်။ အောက်ပါအတိုင်း ရိုက်ပါ - 

        mkdir projects
        ls

mkdir ဆိုတာ "Make Directory" ပါ။ Folder အသစ်ဆောက်တာကို ဆိုလိုပါတယ်။ projects ဆိုတဲ့ folder လေး ဆောက်လိုက်တာပါ။

HTML Structure: <head> နှင့် <body>

<html> element ရဲ့ အထဲမှာ webpage ကို <head> နဲ့ <body> ဆိုပြီး အပိုင်းနှစ်ပိုင်း ထပ်ခွဲထားပါတယ်။

  • <body>: ဒီနေရာကတော့ website မှာ တကယ်ပြမယ့် content တွေ အားလုံး ထည့်ရမယ့် နေရာပါ။ Mock-up ထဲမှာ ပါတဲ့ အရာအားလုံး (စာသားတွေ၊ ပုံတွေ) ကို ဒီထဲမှာပဲ ရေးသွားမှာပါ။
  • <head>: ဒီဘက်မှာတော့ browser က သိထားဖို့ လိုအပ်တဲ့ website ရဲ့ နောက်ကွယ်က အချက်အလက်တွေကို ထည့်ပေးရတာပါ။

ဥပမာ - Browser က ဒီ webpage ကို ဘာနာမည်နဲ့ ခေါ်ရမလဲဆိုတာ သိဖို့အတွက် <head> ထဲမှာ <title> ဆိုတဲ့ element ကို သုံးပါတယ်။ ဒီထဲမှာ "Run Buddy" လို့ ရေးလိုက်ရင် browser ရဲ့ tab ဒါမှမဟုတ် window ပေါ်မှာ အဲ့ဒီနာမည်လေး ပေါ်လာမှာပါ။

စမ်းကြည့်ဖို့အတွက်- HTML ဖိုင်ထဲက title နာမည်ကို ပြောင်းကြည့်ပါ။ ပြီးရင် save လုပ်ပြီး browser မှာ page ကို reload လုပ်ပြီး အပြောင်းအလဲကို ကြည့်လိုက်ပါ။

Assessments (စစ်ဆေးချက်) တွေမှာ အမှတ်ဘယ်လောက်ရလဲဆိုတာ စိတ်မပူပါနဲ့။ ဒါတွေက ကိုယ်သင်ထားတာတွေကို မှတ်မိအောင်နဲ့ ဗဟုသုတ ပိုခိုင်မာလာအောင် ကူညီပေးဖို့ပဲ ဖြစ်ပါတယ်။

  1. <html lang="en">: ဖွင့်ထားတဲ့ <html> tag ထဲမှာ lang="en" လို့ ရေးထားတာကို တွေ့မှာပါ။ ဒါက ဒီ page ရဲ့ အဓိက ဘာသာစကားဟာ အင်္ဂလိပ်စာ ဖြစ်တယ်လို့ browser ကို ပြောပြလိုက်တာပါ။
  2. <meta charset="UTF-8" />: ဒါကို meta tag လို့ ခေါ်ပါတယ်။ Meta tags တွေက browser အတွက် အပိုဆောင်း အချက်အလက်တွေကို ပေးတာပါ။ တချို့က website ပြသပုံအတွက် ဖြစ်သလို၊ တချို့ကတော့ search engine optimization (ရှာဖွေမှုမှာ ထိပ်ဆုံးက ပါလာစေဖို့) အတွက် သုံးပါတယ်။ ဒါတွေအကြောင်းကို နောက်ထပ် modules တွေမှာ အသေးစိတ် ထပ်ပြောသွားပါမယ်။

💡 အပျော်သဘော စမ်းကြည့်ရအောင်

ဒီ emoji (🏃) လေးကို index.html ရဲ့ <body> ထဲမှာ copy/paste လုပ်ကြည့်ပါ။ browser မှာ ပုံမှန်အတိုင်း ပေါ်လာပါလိမ့်မယ်။ ဒါပေမဲ့ meta tag ကို <meta charset="ASCII" /> လို့ ပြောင်းလိုက်ရင်တော့ အလုပ်မလုပ်တော့ပါဘူး။ ဘာလို့လဲဆိုတော့ browser က ASCII character set ထဲမှာပဲ ကန့်သတ်လိုက်လို့ဖြစ်ပြီး အဲ့ဒီထဲမှာ emojis တွေ မပါဝင်လို့ပါ။ ဒါကြောင့် အမြဲတမ်း UTF-8 ကိုပဲ သုံးတာ အကောင်းဆုံးပါ။

သတင်းကောင်းကတော့ ဒါတွေဟာ boilerplate တွေပါပဲ။ ဆိုလိုတာက webpage တိုင်းဟာ ဒီ structure အတိုင်းပဲ စရမှာပါ။ page တိုင်းမှာ DOCTYPE၊ <html> element၊ <head> element စတာတွေ အမြဲတမ်း ပါနေရပါမယ်။ ဒါတွေကို အလွတ်ရအောင် ခဏခဏ ရိုက်ကြည့်တာ အကောင်းဆုံးဖြစ်ပေမဲ့ နောက်ပိုင်း projects အသစ်တွေမှာတော့ copy/paste လုပ်ပြီး သုံးလို့ရပါတယ်။

ကျွန်တော်တို့ index.html ထဲမှာ Code တွေကို ဘယ်လိုစီစဉ်ထားသလဲဆိုတာ သတိထားကြည့်ပါ။ Element အသစ်တစ်ခုတိုင်းကို စာကြောင်းအသစ် (new line) မှာ ရေးတယ်၊ ပြီးတော့ Element တစ်ခုထဲမှာ နောက်ထပ် Element တစ်ခု ထပ်ပါနေရင် Indent (စာကြောင်းအဝင်အထွက်) လုပ်ပေးရပါတယ်။

Code ကို သပ်သပ်ရပ်ရပ် ရေးသားခြင်း

တကယ်တော့ HTML ကို အောက်ကပုံစံအတိုင်း ရေးရင်လည်း Browser က အလုပ်လုပ်ပါတယ် - 

<!DOCTYPE html> <html lang="en"> <head> <meta charset="UTF-8" />
      <title>Run Buddy</title> </head> <body> <h1>RUN BUDDY</h1> What We Do </body>
      </html>

ဒါပေမဲ့ ဒီလိုရေးတာဟာ ဘာတွေရေးထားမှန်း သိဖို့ Sense လုပ်ရတာ တော်တော်ခက်ပါတယ်။ ဒါကြောင့် Code ရေးတဲ့အခါ အမြဲတမ်း Legible ဖြစ်အောင် (ဖတ်ရတာ ရှင်းလင်းနေအောင်) ရေးဖို့ အလေ့အကျင့် လုပ်ထားပါ။

ဒီလိုရေးခြင်းအားဖြင့် -

  • ကိုယ့် Project မှာ ဝိုင်းလုပ်မယ့် တခြား Developers တွေ ဖတ်ရတာ လွယ်ကူစေပါတယ်။
  • နောက်ရက်မှာ ကိုယ့် Code ကို ပြန်ကြည့်တဲ့အခါ ဘာတွေလုပ်ထားသလဲဆိုတာ ချက်ချင်း နားလည်စေပါတယ်။

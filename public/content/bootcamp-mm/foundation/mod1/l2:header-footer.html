1.2
Build the Header and Footer

1.2.1: Introduction

ရှေ့မဆက်ခင် ကိုယ့်ကိုယ်ကိုယ် ချီးကျူးလိုက်ပါဦး! Website တစ်ခုလုံး တည်ဆောက်နိုင်ဖို့အတွက် မင်းအများကြီး ပြင်ဆင်ပြီးသွားပါပြီ။

ပြီးခဲ့တဲ့ သင်ခန်းစာမှာ မင်းလုပ်ခဲ့တဲ့ အရေးကြီးဆုံး အဆင့်တစ်ခုကတော့ content အမျိုးမျိုးကို ထည့်ဖို့အတွက် "containers" (ခေါ်) divs တွေကို တည်ဆောက်ခဲ့တာပါ။ ဒီသင်ခန်းစာမှာတော့ အဲ့ဒီ divs တွေကြောင့် တစ်ခုနဲ့တစ်ခု ရောကုန်မှာ မစိုးရိမ်ရဘဲ section တစ်ခုချင်းစီကို စိတ်ကြိုက် content တွေ၊ style တွေ ဘယ်လိုထည့်မလဲဆိုတာကို တွေ့ရမှာပါ။

အောက်ကပုံကတော့ အခုထိ မင်းတည်ဆောက်ထားတဲ့ ပုံစံပါ -

(အဖြူရောင်နောက်ခံမှာ စာသားတွေပဲ ပါဝင်တဲ့ Run Buddy Layout ပုံ)

ဒီသင်ခန်းစာကနေ ဘာတွေတတ်မြောက်မလဲ?

အခုရှိနေတဲ့ ဒီဇိုင်းကို အခြေခံပြီး အောက်ပါအချက်တွေကို လုပ်ဆောင်သွားမှာပါ -

  • HTML နဲ့ CSS documents တွေကို ချိတ်ဆက်ဖို့ relative paths တွေကို သုံးမယ်။
  • သတ်မှတ်ထားတဲ့ HTML elements တွေကို style သတ်မှတ်ပေးဖို့ CSS selectors တွေကို သုံးမယ်။
  • Webpage တစ်ခုအတွက် header နဲ့ footer components တွေကို တည်ဆောက်မယ်။
  • HTML elements တွေအကြောင်း အသေးစိတ် အချက်အလက်တွေ ဖော်ပြဖို့ HTML attributes တွေကို သုံးမယ်။

1.2.2: Preview

ဒီသင်ခန်းစာမှာ Website ရဲ့ header နဲ့ footer ကို content တွေထည့်ပြီး style ပြင်သွားမှာပါ။ အားလုံးပြီးသွားရင်တော့ landing page က အရင်ကပြထားတဲ့ ပုံစံအတိုင်း ဖြစ်သွားပါလိမ့်မယ်။

(Header နဲ့ Footer ကို Style သွင်းပြီးသား Run Buddy Landing Page ပုံ)

ပုံမှန်အားဖြင့် Website တစ်ခုရဲ့ header နဲ့ footer ဟာ style ပုံစံချင်း တူတတ်တာကြောင့် တစ်ခါတည်း တွဲလုပ်တာက ပိုအဆင်ပြေပါတယ်။ သူတို့ဟာ Website တိုင်းရဲ့ အဓိကကျတဲ့ နေရာတွေ (anchors) ဖြစ်ပြီး branding (အမှတ်တံဆိပ်) နဲ့ navigation (မီနူးများ) ကို ပြသပေးတဲ့ အရေးကြီးတဲ့ အစိတ်အပိုင်းတွေပဲ ဖြစ်ပါတယ်။

ဒါကို လုပ်ဆောင်ဖို့အတွက် အောက်ပါအဆင့်တွေကို လိုက်နာသွားပါမယ် -

  • Header ထဲကို content ထည့်ခြင်း: HTML page ရဲ့ section တစ်ခုချင်းစီအတွက် navigation links တွေကို ထည့်ပါမယ်။
  • CSS basics ကို လေ့လာခြင်း: Cascading Style Sheets (CSS) က HTML ကို ဘယ်လို လှပအောင် လုပ်ပေးလဲဆိုတာနဲ့ သူ့ရဲ့ syntax (ရေးသားပုံစံ) ကို သင်ယူပါမယ်။
  • Style sheet ကို တည်ဆောက်ခြင်း: Run Buddy ရဲ့ အပြင်အဆင်ကို ပြောင်းလဲပေးမယ့် CSS rules တွေ ထည့်ဖို့ style sheet တစ်ခု ဆောက်ပါမယ်။
  • အခြေခံ CSS styles များ ထည့်သွင်းခြင်း: Header အတွက် စတင်ရေးသားပြီး CSS Box Model က element တွေရဲ့ margins, borders နဲ့ padding တွေကို ဘယ်လို ထိန်းချုပ်လဲဆိုတာ လေ့လာပါမယ်။
  • အသေးစိတ် CSS styles များ ထည့်သွင်းခြင်း: Header ကို mock-up ပုံစံအတိုင်း ဖြစ်သွားအောင် အသေးစိတ် CSS rules တွေ သုံးပါမယ်။
  • Footer အတွက် HTML ကို ပြင်ဆင်ခြင်း: Footer ရဲ့ structure ကို HTML နဲ့ ရေးပါမယ်။
  • Footer ကို Design ဆွဲခြင်း: CSS rules တွေကို သုံးပြီး Footer ကို mock-up အတိုင်း လှပအောင် လုပ်ပါမယ်။

ဒါတွေအားလုံးက လက်တွေ့လုပ်ကြည့်မှ ပိုတတ်မှာဖြစ်လို့ အခုပဲ စတင်လိုက်ရအောင်!

1.2.3: Header ထဲကို Content ထည့်ခြင်း

အခု ကျွန်တော်တို့ <header> element ထဲကို ခေတ်မီ Website တိုင်းမှာ ပါလေ့ရှိတဲ့ Branding နဲ့ Navigation ဆိုတဲ့ အရေးကြီးတဲ့ အပိုင်းနှစ်ခုကို စပြီး ထည့်ပါမယ်။

ဒီအစိတ်အပိုင်းနှစ်ခုကို header နဲ့ footer တွေမှာ အမြဲထည့်လေ့ရှိပါတယ်။ ဒါမှ Website ရဲ့ ဘယ်စာမျက်နှာကိုပဲရောက်ရောက် ပုံစံတစ်မျိုးတည်း (uniform) ဖြစ်နေမှာပါ။ ဥပမာ - မင်းရင်းနှီးတဲ့ Website တစ်ခုရဲ့ အပေါ်ဆုံးပိုင်းဟာ ဘယ်စာမျက်နှာသွားသွား ပုံစံတူနေတာကို တွေ့ရပါလိမ့်မယ်။ အဲ့ဒါဟာ branding ပါပဲ။ တခြားစာမျက်နှာကို သွားချင်ရင် ရှာရမယ့် မီနူး (menu) ကလည်း အပေါ် ဒါမှမဟုတ် အောက်မှာပဲ ရှိနေမှာပါ။ အဲ့ဒါကတော့ navigation ပါ။

Add Branding (အမှတ်တံဆိပ် ထည့်ခြင်း)

Branding ဆိုတာ ကုမ္ပဏီတစ်ခုရဲ့ logo၊ အရောင်တွေနဲ့ အထွေထွေဟန်ပန် (brand) ကို ဆိုလိုတာပါ။ ဥပမာ - Amazon ရဲ့ လိမ္မော်ရောင်မြှားကို မြင်တာနဲ့ ဒါ Amazon ပဲဆိုတာ သိနိုင်သလိုမျိုးပေါ့။

ကျွန်တော်တို့ Lesson 1 တုန်းက <header> ထဲမှာ <h1>RUN BUDDY</h1> ထည့်ခဲ့တာဟာ branding ကို စတင်လုပ်ဆောင်ခဲ့တာပါပဲ။ ဒါကြောင့် Website ရဲ့ အပေါ်ဆုံးမှာ Run Buddy ဆိုတဲ့ နာမည်ကို အမြဲပေါ်နေစေမှာ ဖြစ်ပါတယ်။ ဒီသင်ခန်းစာရဲ့ နောက်ပိုင်းမှာတော့ ဒါကို CSS style တွေနဲ့ ပိုလှအောင် ပြင်သွားပါမယ်။

Add Navigation

Branding အတွက် Content တွေ ထည့်ပြီးပြီဆိုတော့ Navigation (မီနူး) ပိုင်းကို သွားရအောင်။

VS Code ထဲမှာ index.html ကိုဖွင့်ပြီး <h1> ရဲ့ အောက်၊ ပိတ်ထားတဲ့ </header> tag ရဲ့ အပေါ်မှာ <nav> နဲ့ </nav> ကို ထည့်ပေးပါ။

ရေးပြီးရင် အောက်က HTML ပုံစံအတိုင်း ဖြစ်နေရပါမယ် -

    <header>
    <h1>RUN BUDDY</h1>
    <nav></nav>
    </header>

<nav> element ဆိုတာ HTML5 ကနေ စတင်မိတ်ဆက်ပေးခဲ့တဲ့ semantic element တစ်ခုပါ။ Semantic elements တွေဟာ သူတို့ထဲမှာ ဘယ်လိုအချက်အလက်တွေ ပါဝင်လဲဆိုတာကို အမည်အားဖြင့် ဖော်ပြပေးနိုင်တဲ့အတွက် Accessibility နဲ့ SEO အတွက် အများကြီး အထောက်အကူဖြစ်စေပါတယ်။

  • Accessibility: ဒါကတော့ အမြင်အာရုံ ဒါမှမဟုတ် အကြားအာရုံ ချို့တဲ့သူတွေအတွက် screen readers လိုမျိုး အထောက်အကူပြု နည်းပညာတွေကနေ Website ရဲ့ အကြောင်းအရာကို သေသေချာချာ ရှင်းပြနိုင်အောင် HTML elements တွေကို စနစ်တကျ ရေးသားတဲ့ နည်းလမ်းပါ။
  • SEO (Search Engine Optimization): Website ကို လူကြည့်များအောင်နဲ့ Google လိုမျိုး search engine တွေမှာ ရှာလိုက်ရင် ထိပ်ဆုံးမှာ ပေါ်လာအောင် လုပ်တဲ့ လုပ်ငန်းစဉ်ပါ။

DEEP DIVE

Website တွေမှာ Accessibility ကောင်းအောင် လုပ်ဖို့ နည်းလမ်းတွေ အများကြီး ရှိတဲ့အထဲက အလွယ်ဆုံးကတော့ HTML5 ရဲ့ semantic elements တွေကို သုံးတာပါပဲ။ HTML5 မတိုင်ခင်တုန်းကတော့ Website တစ်ခုလုံးမှာရှိတဲ့ Content တွေ အကုန်လုံးအတွက် <div> element တစ်ခုတည်းကိုပဲ သုံးခဲ့ကြပါတယ်။ ဒါကြောင့် Developers တွေဟာ screen readers တွေ အဓိပ္ပာယ်ဖော်နိုင်ဖို့ အပင်ပန်းခံပြီး သီးသန့် လုပ်ဆောင်ပေးခဲ့ရပါတယ်။

အခုတော့ <header>, <footer>, နဲ့ <nav> တို့ဟာ <div> လိုပဲ အလုပ်လုပ်ပေမဲ့ သူတို့ထဲမှာ ဘာတွေပါဝင်လဲဆိုတာကို Browser ကို တိုက်ရိုက်ပြောပြနိုင်တဲ့ အဓိပ္ပာယ်ရှိတဲ့ နာမည်တွေ ပါဝင်လာပါတယ်။

Navigation ကို List နဲ့ စီစဉ်ခြင်း

Navigation ကို စီစဉ်တဲ့နေရာမှာ link တွေကို စာရင်း (list) အနေနဲ့ ပြတာက လူသုံးအများဆုံး နည်းလမ်းတစ်ခုပါ။

မင်းစောစောက ဆောက်ခဲ့တဲ့ <nav> tag ရဲ့ အနောက်မှာ အောက်က code တွေကို ရိုက်ထည့်လိုက်ပါ -

    <!-- Unordered list element -->
    <ul>
    <!-- List item element-->
    <li>
    <!-- Anchor element -->
    <a href="#what-we-do">What We Do</a>
    </li>
    <li>
    <a href="#what-you-do">What You Do</a>
    </li>
    <li>
    <a href="#your-trainers">Your Trainers</a>
    </li>
    <li>
    <a href="#reach-out">Reach Out</a>
    </li>
    </ul>

အခုရေးလိုက်တဲ့ Code တွေက ဘာတွေလဲဆိုတာ တစ်ခုချင်းစီ ခွဲကြည့်ရအောင်။

HTML ရဲ့ List အမျိုးအစားများ

HTML မှာ စာရင်း (List) ပြချင်ရင် အမျိုးအစား ၂ ခု ရှိပါတယ် -

  • Ordered lists (`<ol>`): နံပါတ်စဉ်တွေ (1, 2, 3) နဲ့ အစဉ်လိုက် ပြချင်တဲ့အခါ သုံးပါတယ်။
  • Unordered lists (`<ul>`): နံပါတ်စဉ် မလိုဘဲ အစက်လေးတွေ (bullet points) ဒါမှမဟုတ် လေးထောင့်ကွက်လေးတွေနဲ့ပဲ စာရင်းပြချင်ရင် သုံးပါတယ်။

ဒီ List တွေထဲမှာ ပါဝင်မယ့် အချက်အလက် တစ်ခုချင်းစီကိုတော့ List item (`<li>`) tag နဲ့ ရေးရပါတယ်။

Parent/Child Relationship (မိဘနှင့် သားသမီး ဆက်နွယ်မှု)

HTML မှာ Parent/Child relationship ဆိုတာ ရှိပါတယ်။

  • <ul> ဒါမှမဟုတ် <ol> က Parent (မိဘ) ဖြစ်ပါတယ်။
  • သူ့ထဲမှာ ထည့်ထားတဲ့ <li> ကတော့ Child (သားသမီး) ဖြစ်ပါတယ်။

ဆိုလိုတာက <li> (သားသမီး) တွေကို Parent ရဲ့ tag အဖွင့်နဲ့ အပိတ်ကြားထဲမှာပဲ စနစ်တကျ ထည့်သွင်း (nest) ရမှာဖြစ်ပါတယ်။

Anchor Elements () နှင့် Attributes

<li> element ထဲမှာ ကြိုက်တဲ့အရာကို ထည့်လို့ရပါတယ်။ အခုကတော့ တစ်နေရာကနေ တစ်နေရာကို ချိတ်ဆက်ပေးမယ့် Anchor element (<a>) ကို သုံးပြီး Links တွေ ဖန်တီးလိုက်တာပါ။ ဒီ link တွေကို နှိပ်လိုက်ရင် အောက်ပါနေရာတွေကို ရောက်သွားနိုင်ပါတယ် -

  • ဒီစာမျက်နှာထဲက တခြားနေရာတစ်ခု။
  • ကိုယ့် website ထဲမှာရှိတဲ့ တခြား HTML page တစ်ခု။
  • တခြား website တစ်ခုခု (ဥပမာ - Google)။

HTML (Hypertext Markup Language) ရဲ့ "Hypertext" ဆိုတာ တခြားဆီကို ချိတ်ဆက်ပေးထားတဲ့ စာသားကို ပြောတာပါ။ ဒါကြောင့် <a> tag ဟာ HTML ရဲ့ အနှစ်သာရတစ်ခုလို့ ပြောလို့ရပါတယ်။

အောက်က Code နမူနာကို ကြည့်ပါ -

    <!-- Google ကို သွားမယ့် link -->
    <a href="https://google.com">Go to Google</a>
    
    View Our Privacy Policy

Attributes (ဂုဏ်သတ္တိများ)

အပေါ်က code မှာ href= ဆိုတာကို သတိထားမိမှာပါ။ ဒါကို HTML Attribute လို့ ခေါ်ပါတယ်။ Attributes တွေဟာ HTML element တစ်ခုကို -

  • သီးသန့် နာမည်ပေးဖို့ (Identity)။
  • တခြား element တွေနဲ့ ချိတ်ဆက်ဖို့။
  • CSS သုံးပြီး ဒီဇိုင်းပြင်ဖို့အတွက် သုံးပါတယ်။

အချို့ element တွေအတွက် attributes တွေက မရှိမဖြစ် လိုအပ်ပါတယ်။ ဥပမာ - <a> tag မှာ href (သွားရမယ့် လိပ်စာ) မပါရင် link က အလုပ်လုပ်မှာ မဟုတ်ပါဘူး။

အသုံးအများဆုံး Attributes များမှာ -

  • id: Element တစ်ခုချင်းစီကို ပေးတဲ့ သီးသန့်နာမည်ပါ။ Page တစ်ခုလုံးမှာ တစ်ခုပဲ ရှိရပါမယ်။ (Unique identifier)
  • class: Element အုပ်စုလိုက်ကို နာမည်ပေးတာပါ။ ပုံစံတူချင်တဲ့ နေရာတွေမှာ နာမည်တစ်ခုတည်းကို အကြိမ်ကြိမ် ပြန်သုံးလို့ ရပါတယ်။
  • title: Mouse ကို link ပေါ် တင်ထားလိုက်တဲ့အခါ ပေါ်လာမယ့် စာတိုလေး (Tool-tip) ကို သတ်မှတ်တာပါ။

HTML attributes တွေအကြောင်း ပိုသိချင်ရင် MDN Web Docs မှာ သွားရောက်ဖတ်ရှုနိုင်ပါတယ်။ ဒီစာမျက်နှာကို Bookmark လုပ်ထားဖို့လည်း အကြံပြုပါရစေ။

Internal Links (စာမျက်နှာအတွင်း ခုန်ကူးခြင်း)

ကျွန်တော်တို့ Navigation မှာ ထည့်ခဲ့တဲ့ href attributes တွေက လက်ရှိ page ထဲမှာတင် တစ်နေရာကနေ တစ်နေရာကို ခုန်ကူး (Jump) ဖို့ သုံးတာပါ။

  • href="#what-we-do" လို့ ရေးလိုက်ရင် Browser က အဲ့ဒီ <a> ကို နှိပ်လိုက်တဲ့အခါ စာမျက်နှာထဲမှာ id="what-we-do" လို့ ပေးထားတဲ့ Element ကို လိုက်ရှာပါလိမ့်မယ်။
  • မှတ်ထားရမှာက Link ချိတ်တဲ့အခါ Hashtag (#) လေးကို နာမည်ရှေ့မှာ ထည့်ပေးရမှာပါ။ (အဲ့ဒီ id တွေကိုတော့ မကြာခင်မှာ ထည့်သွားပါမယ်။)

Branding ကို Link ချိတ်ခြင်း

Website တိုင်းမှာ လုပ်လေ့ရှိတဲ့ Best Practice တစ်ခုကတော့ Website ရဲ့ Branding/Logo ကို နှိပ်လိုက်ရင် Homepage ကို ပြန်ရောက်သွားအောင် လုပ်တာပါ။ ဒါကြောင့် <h1> ထဲက စာသားကို အောက်ပါအတိုင်း <a> နဲ့ ပတ်လိုက်ရအောင် -

    <h1>
    <a href="/">RUN BUDDY</a>
    </h1>

ဒီမှာ သုံးထားတဲ့ Forward slash ("/") ဆိုတာ Project တစ်ခုလုံးရဲ့ အမြင့်ဆုံး Directory (Main Folder) ကို ကိုယ်စားပြုပါတယ်။ အသုံးပြုသူက ဒါကို နှိပ်လိုက်ရင် Main Folder ဆီကို ရောက်သွားမှာဖြစ်ပြီး၊ ဖိုင်နာမည် သီးသန့်မပါတဲ့အတွက် index.html ဖိုင်ကပဲ အလိုအလျောက် ပွင့်လာမှာ ဖြစ်ပါတယ်။

REWIND

အရင်သင်ခန်းစာမှာ ပြောခဲ့တာကို ပြန်သတိရကြည့်ပါ။ ကွန်ပျူတာဟာ ဖိုင်နာမည် တိတိကျကျ မပါရင် "index" page ကိုပဲ အရင်ဆုံး ရှာဖွေပေးတာပါ။ ဒါကြောင့်ပဲ Web development မှာ Homepage ရဲ့ ဖိုင်နာမည်ကို index.html လို့ ပေးရတာ ဖြစ်ပါတယ်။

အခုဆိုရင် မင်းရဲ့ <header> ထဲက Code တွေဟာ အောက်ပါအတိုင်း ဖြစ်နေပါပြီ။ မင်းတို့ အဆင်သင့်ဖြစ်ပြီဆိုရင် ဒီ Landing page ကို လှပအောင် စပြီး ပြင်ဆင်ကြရအောင်!

အခုဆိုရင် မင်းရဲ့ <header> ထဲမှာ Branding (စာသား link) နဲ့ Navigation (မီနူးစာရင်း) တွေ ပြည့်ပြည့်စုံစုံ ပါသွားပါပြီ။ Code ပုံစံကတော့ အောက်ပါအတိုင်း ဖြစ်နေရပါမယ် -

Commit to Git and Push to GitHub

အလုပ်တွေ တော်တော်လေး လုပ်ပြီးသွားပြီဖြစ်လို့ အခုလုပ်ထားတာတွေကို မပျောက်အောင် GitHub ပေါ် Push လုပ်ပြီး သိမ်းလိုက်ရအောင်။

Git Commands များ ပြန်လည်နွေးထည့်ခြင်း

Terminal သို့မဟုတ် Git Bash ကိုဖွင့်ပြီး အောက်ပါ commands တွေကို အစဉ်လိုက် ရိုက်ထည့်ပါ -

  1. git add -A သို့မဟုတ် git add . :

    ပြင်ဆင်လိုက်တဲ့ Code တွေနဲ့ ဖိုင်အသစ်တွေကို Git က မှတ်သားနိုင်အောင် လုပ်တာဖြစ်ပါတယ်။

  2. git commit -m "add navigation and branding links" :

    ဘာတွေ ပြောင်းလဲလိုက်လဲဆိုတာကို note ရေးပြီး မှတ်တိုင် (save point) တစ်ခု အနေနဲ့ သိမ်းလိုက်တာပါ။

  3. git push origin main :

    ကိုယ့်စက်ထဲက Code တွေကို GitHub remote server ပေါ် တင်လိုက်တာ ဖြစ်ပါတယ်။

ဒါဆိုရင် မင်းရဲ့ အလုပ်တွေဟာ အင်တာနက်ပေါ်မှာ လုံလုံခြုံခြုံ ရှိသွားပါပြီ!

SHOW HINT

git add နဲ့ git commit command နှစ်ခုဟာ locally မှာ ကိုယ်ဘာတွေသိမ်းမယ်ဆိုတာနဲ့ ဘာကြောင့်သိမ်းတယ်ဆိုတာကို Git ကို အရင်ပြောပြတာပါ။ GitHub ပေါ် push မလုပ်ခင် ဒီအဆင့်နှစ်ခုကို (ဒီအတိုင်းအစဉ်လိုက်) အရင်ဆုံး လုပ်ရပါမယ်။

  • git push origin main: မင်းရဲ့ နောက်ဆုံး commit ထဲက code တွေနဲ့ ဖိုင်အပြောင်းအလဲ အားလုံးကို GitHub ပေါ် တင်ပေးမယ့် command ပါ။

အခုဆိုရင် မင်းရဲ့ code တွေဟာ ကိုယ့်စက် (locally) မှာရော GitHub (remotely) မှာပါ လုံခြုံသွားပါပြီ။ မင်းရဲ့ GitHub Pages link ဖြစ်တဲ့ <username>.github.io/run-buddy မှာ သွားကြည့်ရင် Browser ထဲမှာ HTML အသစ်တွေ ဘယ်လိုပေါ်နေလဲဆိုတာ မြင်ရပါလိမ့်မယ်။

ကျွန်တော်တို့ href တွေမှာ # သင်္ကေတကို သုံးခဲ့တာ သတိထားမိမှာပါ။ ဒါကို programmer တွေက octothorpe လို့ ခေါ်ကြပါတယ်။ ဒါဟာ စာမျက်နှာတစ်ခုထဲမှာတင် သတ်မှတ်ထားတဲ့ နေရာကို ခုန်ကူးဖို့ သုံးတာပါ။

Website ရဲ့ အသွင်အပြင် (Looks Matter)

အခုထိ ကျွန်တော်တို့ သုံးခဲ့တဲ့ HTML elements တွေက အဓိက အချက်နှစ်ချက်ကိုပဲ လုပ်ပေးနိုင်ပါသေးတယ် -

  • Organizing content: အချက်အလက်တွေကို စနစ်တကျ စီစဉ်ပေးတာ (ဥပမာ- <h1> tags ကြားထဲက စာသားများ)။
  • Providing interactivity: Interactivity (အပြန်အလှန် တုံ့ပြန်မှု) ပေးတာ (ဥပမာ- link တွေကို နှိပ်ရင် တခြားနေရာရောက်အောင် လုပ်ပေးတာ)။

ဒါပေမဲ့ အရေးကြီးတာ တစ်ခု ကျန်ပါသေးတယ်၊ အဲ့ဒါက Website ကို ကြည့်ကောင်းအောင် (Look Good) လုပ်ဖို့ပါ။

အရင်တုန်းကတော့ Website ရဲ့ ဒီဇိုင်းပိုင်းကိုပါ HTML ကပဲ အကုန်လုပ်ခဲ့ရတာပါ။ အဲ့ဒီတုန်းက Developers တွေမှာ ရွေးချယ်စရာ သိပ်မရှိပါဘူး။ စာလုံးအရောင်ပြောင်းတာ၊ နောက်ခံအရောင်ထည့်တာနဲ့ ပုံတွေကို အကြီးအသေးလုပ်တာလောက်ပဲ လုပ်လို့ရပါတယ်။

ခက်ခဲတဲ့ ဒီဇိုင်းတွေဆိုရင် ပုံတစ်ပုံလုံးကို အပိုင်းအစလေးတွေ အများကြီး ညှပ်၊ ပြီးမှ HTML table element ထဲမှာ ပြန်စီပြီး Website ပေါ်မှာ ပုံဖော်ခဲ့ရတာပါ။ ဒါဟာ Microsoft Excel ရဲ့ ဆဲလ်ကွက်တွေထဲမှာ ပုံစုတ်လေးတွေကို လိုက်စီနေရသလိုမျိုး အရမ်းကို ပင်ပန်းတဲ့ အလုပ်ပါ။

LEGACY LORE

အရင်ခေတ်က Website ဒီဇိုင်းတွေက အခုခေတ်နဲ့ယှဉ်ရင် အရမ်းကို ကြည့်ရဆိုးပါတယ်။ ဒါကြောင့်ပဲ ခေတ်မီတဲ့ Web design တွေအတွက် Best practices (အကောင်းဆုံး လုပ်ဆောင်နည်းတွေ) ပေါ်ပေါက်လာရတာပါ။

Developer တွေက ဒီဇိုင်းကို ပိုပြီး ထိန်းချုပ်ချင်လာတဲ့အခါ HTML တစ်ခုတည်းနဲ့ မလုံလောက်တော့ဘူးဆိုတာ သိလာကြပါတယ်။ ဒါကြောင့် HTML ဆီက ဒီဇိုင်းပိုင်းဆိုင်ရာ ဝန်ထုပ်ဝန်ပိုးတွေကို ခွဲဝေယူဖို့ Language အသစ်တစ်ခု ပေါ်လာခဲ့ပါတယ်၊ အဲ့ဒါကတော့ CSS ပဲ ဖြစ်ပါတယ်။
